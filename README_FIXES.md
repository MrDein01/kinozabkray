# Правки v16 — загрузка фото новостей на Vercel

На Vercel нельзя надёжно сохранять загруженные файлы в `public/uploads`, потому что это serverless-деплой, а файловая система после деплоя не работает как обычный VPS.

## Что исправлено

1. `app/api/upload/route.ts` больше не пытается записывать файл в `public/uploads`.
2. Маленькое изображение конвертируется в `data URL` и возвращается в админку.
3. После нажатия `Сохранить` картинка сохраняется в поле `imageUrl` новости в базе Neon.
4. Ограничение размера изображения: до 4 МБ.

Это самый простой бесплатный вариант для Vercel без подключения отдельного файлового хранилища.

## Как применить

Распакуйте архив в корень проекта с заменой файлов, затем выполните:

```cmd
cd C:\Users\User\Desktop\kino75-main
pnpm build
git add -A
git commit -m "Fix news image upload on Vercel"
git push
```

После `git push` Vercel автоматически начнёт новый деплой.

## Важно

Если нужны большие фотографии и много новостей, лучше позже подключить Vercel Blob, Supabase Storage или Cloudinary. Этот фикс сделан как быстрый и бесплатный вариант.
